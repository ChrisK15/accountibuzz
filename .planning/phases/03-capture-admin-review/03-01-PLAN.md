## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local-dev → installed dev binary | The plugin block in source declares permissions; the installed binary actually requests them at runtime. Mismatch (binary lacks permissions but source declares them) is the failure mode this plan prevents via the rebuild checkpoint. |
| package registry → installed node_modules | `expo install` resolves a version pin from the Expo SDK 55 compatibility list — trusted source. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-01 | Tampering | `app.config.ts` plugin block | mitigate | Use the EXACT permission strings from RESEARCH (verified against project copywriting voice). Manual `infoPlist` / `android.permissions` arrays are FORBIDDEN — they would conflict with the plugin's generated entries and silently override our copy with system defaults. |
| T-03-01-02 | Information Disclosure | iOS / Android OS permission prompt copy | mitigate | The two prompt strings declare exactly why the app needs camera + mic — no leak of internal IDs, no scope creep beyond "capture your daily proof." Strings reviewed against Phase 1 voice rules (warm, energetic, concise — never corporate). |
| T-03-01-03 | Denial of Service | Dev client install lifecycle | accept | The rebuild step is a one-time human action per device; if the user skips it, capture silently fails on first launch. | T-03-01-04 | Elevation of Privilege | netinfo native module supply chain | accept | `@react-native-community/netinfo` is a community-maintained but widely-used React Native ecosystem package; no recent CVEs at version `^11.x`. Supply-chain risk equivalent to every other RN dep already in the tree. |
