# Phase 2 — UI Design Contract

> Visual and interaction contract for Groups & Invites.
> Generated by gsd-ui-researcher. Extends Phase 1 contract (`01-UI-SPEC.md`) — no new visual tokens, only new screens, patterns, and three component additions.

---

## Relationship to Phase 1

**Phase 1 is the source of truth for all visual tokens.** This contract inherits — unchanged — Phase 1's:

- Design system (hand-rolled RN primitives, no shadcn, Manrope font, `@expo/vector-icons`)
- Full color palette (yellow `#FFDE42` primary, cyan `#53CBF3` accent, `hsl(4 78% 56%)` destructive, warm off-white / near-black dual theme)
- Full typography scale (Display 32/800, H1 24/700, H2 20/700, Body 16/500, Caption 13/500) — see **Typography §Locked Exceptions** below for justification of the 5-size / 3-weight scale
- Full spacing scale (4 / 8 / 12 / 16 / 24 / 32) — includes 12px (user-locked exception documented in Phase 1)
- All radii (6 / 12 / 20 / pill) and elevations (e1 / e2)
- All button, input, and avatar primitives
- Copywriting voice: **warm, energetic, concise — never corporate.**

**Phase 2 adds to, never overwrites, this contract.** If a token feels missing, use the closest P1 token before inventing one. Any deviation must be justified in a new row below and explicitly supersedes P1.

## Typography

**Inherited from Phase 1 — unchanged.**

### Locked Exceptions (user decision, documented so future checkers don't re-flag)

Phase 1's typography scale contains **5 sizes** (Display 32, H1 24, H2 20, Body 16, Caption 13) and **3 weights** (500 Medium, 700 Bold, 800 ExtraBold). The standard design-quality heuristic caps these at 4 sizes and 2 weights respectively. Both values are **user-locked decisions sourced from `design_refs/design_token_2.png` (Lovable-generated, user-supplied 2026-04-21)** — the same origin as the user-locked 12px spacing exception.

| Deviation | Standard cap | Actual value | Justification |
|-----------|--------------|--------------|---------------|
| Font sizes | 4 | **5** (Display 32, H1 24, H2 20, Body 16, Caption 13) | User-supplied Lovable tokens; Phase 1 is already shipped against this scale. Each role has distinct usage: Display for screen heroes, H1 for section headers, H2 for modal titles / secondary headers, Body for UI/copy, Caption for metadata. Collapsing would force either hero-to-header flattening or metadata/body collision — both visible regressions vs. the reference design. |
| Font weights | 2 | **3** (500 Medium, 700 Bold, 800 ExtraBold) | Manrope is a variable font — the third weight costs zero at runtime (no extra font file). User-supplied Lovable tokens reserve 800 specifically for Display-size hero titles, where 700 alongside 32px size reads thin against warm-off-white / near-black backgrounds at device scale. 500/700 carry all UI + heading hierarchy; 800 is Display-only. |

**This is an explicit override of the dimension-4 checker defaults.** The rule does not apply because the scale was user-locked in Phase 1 and ratified by the Phase 1 checker (analogous to the 12px spacing exception). Phase 2 inherits without modification.

### P2 roles that reuse the inherited scale

| P2 Surface | Role Applied | Rationale |
|-----------|--------------|-----------|
| Group name on detail header | Display (32/800) | Matches screen-hero treatment of auth screens |
| Group name in list row | Heading 2 (20/700) | Secondary prominence, dense list |
| Goal description in list row | Body (16/500), **2-line truncate** with `numberOfLines={2}` + ellipsis | Preview, not full text |
| Member display name | Body (16/500) | Matches profile treatment |
| "ADMIN" badge label | Caption (13/500), uppercase, `letterSpacing: 0.5` | Distinguishes without size escalation |
| Invite code (`ABCD-EF12`) | **Body 16 + `fontFamily: 'Manrope_700Bold'` + `fontVariant: ['tabular-nums']` + `letterSpacing: 2`** | Monospace-feel without shipping a mono font; legibility for manual entry |
| Code-entry input | Same treatment as invite code display (tabular-nums, letterSpacing, 700-bold, 20pt size) | Visual continuity with what's being typed |
| Empty-state heading | Heading 1 (24/700) | Below display, above body — inviting but not shouting |
| Modal title | Heading 2 (20/700) | Inherited from P1 "Heading 2 — modal titles" |
| Member count metadata | Caption (13/500), `--text-muted` | "5 of 10 members" |

**No new font roles.** No new weights.

## Color

**Inherited from Phase 1 — unchanged palette.** New reserved-for assignments:

### Accent Reserved-For List (extension)

`--accent` (cyan `#53CBF3`) continues to be **links-and-focus only**. P2 additions:

1. "Join with code" text-link on groups-list empty state (sibling to primary CTA)
2. "Regenerate code" text-link inside admin invite panel
3. "Copy" inline text-link next to the invite code
4. "Transfer admin" text-link in the admin-leave modal
5. Focused border on the code-entry input and goal-description textarea (same 2px ring as P1)

`--accent` is still NEVER used for: primary CTAs, body text, icons by default, backgrounds, badges, success confirmations.

### Destructive Reserved-For List (extension)

`--destructive` (red) extension for P2:

1. "Leave group" text button (bottom of group-detail, non-admin members)
2. "Delete group" text button (bottom of group-detail, admin only — shown after Transfer-admin in the branching leave flow)
3. Confirmation-dialog destructive CTA for all three: Leave / Delete / Regenerate (regenerate is *warning-destructive* — it invalidates an in-flight invite)
4. Typed-error inline copy for `group_full`, `invite_expired`, `invite_already_used`, `already_member`

### Primary Reserved-For List (extension)

`--primary` (yellow `#FFDE42`) continues as the *primary CTA filled-button* color. P2 additions:

1. "Create group" button (groups-list empty state AND `/groups/new` submit)
2. "Join group" button (code-entry screen AND deep-link preview screen post-auth)
3. "Share code" button (admin invite panel — opens native share sheet)
4. "Transfer admin" confirmation CTA (inside Transfer-admin picker modal, after a recipient is picked)
5. "ADMIN" badge background on member rows uses **`--primary` at full saturation** with `--primary-fg` text (matches logo "buzz" pill treatment — declared here as the first non-button use of `--primary`, intentional because admin designation is a core trust signal and deserves the brand color)

|---------|-------|-------|

### 60 / 30 / 10 Split (P2)

- **60% dominant** → `--background` (unchanged)
- **30% secondary** → `--surface` cards (group-list rows, admin panel, member list) + `--surface-muted` (code chip, segmented-control track)
- **10% accent** → `--primary` (CTAs + admin badge) and `--accent` (links/focus) — still combined to ~10%. Admin-badge usage is the only new accent surface; it appears at most once per member row and is visually contained.

## Copywriting Contract

Voice reminder: **warm, energetic, concise — never corporate.** Use *we* / *your group* / *friends*, not *user* / *organization*.

**Dismiss-label rule (applies to every modal in this phase):** The word `Cancel` is banned as a dismiss label. Every modal ships with a context-specific dismiss label that tells the user what they are *not* doing (e.g. `Stay in group`, `Keep my admin role`). This is enforced in the `Modal` primitive — `cancelLabel` has **no default**; callers must pass a context-specific string.

|--------|-------------|------|----------------|---------|

| Admin: Leave group (branching) | `Admins can't just leave` | `Hand the group to someone else, or delete it entirely.` | `Transfer admin instead` (primary) | Secondary `Delete the group` (destructive text, stacked below primary) + `Never mind` (ghost) |
| Admin: Transfer admin (picker) | `Pick a new admin` | `They'll take over the group. You'll stay as a member.` | `Transfer to {name}` (primary, disabled until a member is selected) | `Keep my admin role` (ghost) |
| Admin: Delete group | `Delete this group?` | `Everyone loses access. Submissions, members, and invites are gone. This can't be undone.` | `Delete group` (destructive filled) | `Keep the group` (ghost) |
| Admin: Regenerate invite | `Make a new code?` | `The current code stops working right away. Anyone who hasn't joined yet will need the new one.` | `Regenerate` (primary — recoverable) | `Keep current code` (ghost) |

## Component Additions (Phase 2)

Three new primitives. All obey Phase 1 tokens. All live in `src/components/`.

### 1. `SegmentedControl`

| Prop | Type | Notes |
|------|------|-------|
| `options` | `Array<{ value: string; label: string }>` | Exactly 2 for P2 (Photo / Video); architecture supports 2-4 |
| `value` | `string` | Controlled |
| `onChange` | `(value: string) => void` | — |
| `disabled` | `boolean` | Greys track + text to `--text-muted` |

Visuals: track = `--surface-muted`, radius `md`; inactive segment = transparent, text `--text-muted`; active segment = `--surface`, 1px `--border`, `e1-subtle` shadow, text `--text-strong`, weight 700 (bold). Height 44pt. Equal-width segments via `flex: 1`. Press animation matches PrimaryButton (scale 0.97 / opacity 0.9, 150ms).

### 2. `InviteCodeChip`

| Prop | Type | Notes |
|------|------|-------|
| `code` | `string` | 8-char raw (no dashes); rendered chunked as `XXXX-XXXX` |
| `onCopy` | `() => void` | Triggers `Clipboard.setStringAsync` + haptic + success-text swap |

Visuals: background `--surface-muted`, 1px `--border`, radius `md`, vertical padding `lg`, horizontal padding `xl`. Code rendered with body-16 size bumped to **20pt, weight 700, `letterSpacing: 2`, `fontVariant: ['tabular-nums']`**. Chip is visually centered. Right-aligned `Copy` accent-cyan text button sits on the same row (or below on narrow screens). Long-press the chip → native copy menu (RN `Pressable` + `Clipboard`).

### 3. `Modal` (centered sheet primitive)

| Prop | Type | Notes |
|------|------|-------|
| `visible` | `boolean` | — |
| `onDismiss` | `() => void` | Called on scrim tap and dismiss button |
| `title` | `string` | Heading 2, centered |
| `body` | `ReactNode` | Body-16, `--text`, centered |
| `primaryAction` | `{ label: string; onPress: () => void; variant: 'primary' \| 'destructive'; loading?: boolean }` | Full-width button |
| `secondaryAction?` | `{ label: string; onPress: () => void; variant: 'destructive-text' \| 'ghost' }` | Optional stacked below primary |
| `cancelLabel` | `string` | **No default — caller must supply a context-specific dismiss label** (e.g. `Stay in group`, `Keep the group`). The generic word `Cancel` is disallowed by the phase copywriting contract; the component throws a dev-mode warning if the supplied label is `Cancel` (case-insensitive). |

Visuals: scrim `rgba(0,0,0,0.45)` light / `rgba(0,0,0,0.65)` dark. Sheet centered, max width 400pt, `lg` radius, padding `xl`, `e2-raised` elevation in light / 1px `--border` in dark. Content stack gap `lg`. Buttons stack full-width, `lg` gap between primary and secondary. Dismiss button centered below at `2xl` margin.

Additional primitives that **do not need new components** — reuse P1:

- **Banner** (post-create) → compose with `ScreenHeader` siblings: a `View` with `--surface-muted` bg, 1px border, body-16 text, dismiss × via `Pressable` with `@expo/vector-icons/Feather` `x` icon. Ships inline in the group-detail screen, not extracted as a shared component.
- **Admin badge pill** → inline `View` with `--primary` bg + `--primary-fg` caption-size uppercase text, `pill` radius, `xs`/`sm` padding. Inline in member-row render.
- **Member row** → inline composition of `Avatar` + name stack + badge. Not extracted.

## Screen-by-Screen Contract

### `app/(app)/index.tsx` — Groups list (signed-in home)

**Two visual states.**

**Empty state:**
Nav bar: title `Your groups` left, **Avatar (profile shortcut, tappable)** right (tap → `/profile`) → Screen header (Display "No groups yet" + body subtitle) → Vertical stack (centered): `Create a group` PrimaryButton → 16pt gap → `Join with a code` accent-cyan text link.

**Populated state:**
Nav bar: title `Your groups` left, `+` icon (Feather `plus`) right (tap → `/groups/new`), kebab (Feather `more-horizontal`) right-rightmost containing menu item `Join with a code` (tap → `/groups/join`) + `Profile` (tap → `/profile`). Avatar shortcut only in empty state — moves to kebab in populated state to make room for `+`.
Body: FlatList of group rows. Each row tappable → `/groups/{id}`. Row layout per §Color "Group-list row" above. Pull-to-refresh enabled via TanStack Query `refetch`.

**Loading state (initial):** Skeleton rows (3 of them) with `--surface-muted` blocks matching the row layout. No spinner overlay. Subsequent refetches render silently.

### `app/(app)/groups/new.tsx` — Create group

Nav bar: `← Back` (back chevron) left, title `Start a group` center-left (H2). Form fields in order:

1. `Group name` — TextInput
2. `What's the daily goal?` — TextInput with `multiline`, `numberOfLines={3}`, char counter below (right-aligned caption, turns destructive at 0 or ≥140)
3. `What do you post?` — SegmentedControl (`Photo` | `Video`), default `Photo`
4. `Group timezone` — Read-only TextInput (disabled styling), right-side chevron icon (Feather `chevron-right`), tap opens `IanaTimezonePicker` modal. Default populated via `Intl.DateTimeFormat().resolvedOptions().timeZone` with human label resolved client-side (`Intl.DateTimeFormat('en', { timeZoneName: 'long' })` parsing).
5. Sticky-bottom PrimaryButton `Create group` (disabled until RHF validates).

**Loading state:** Button shows spinner (P1 pattern). Form disabled during submit.

**Post-success behavior:** `router.replace('/groups/{id}')` to the new group's detail screen. Detail screen shows one-time banner (see below).

### `IanaTimezonePicker` (modal, invoked from create-group)

Full-screen modal (`presentationStyle="pageSheet"` on iOS, standard modal on Android). Header: close × left, title `Pick a timezone` center. Searchable list: TextInput at top (placeholder `Search cities or regions`), FlatList below of `{ iana: 'America/Los_Angeles', label: 'Pacific Time — Los Angeles' }` entries. Row tap → close modal, return selected. Empty search → `No match. Try another city or region.` centered in body. IANA source is the platform's `Intl.supportedValuesOf('timeZone')` (RN 0.83 / Hermes supports it) — no third-party library.

### `app/(app)/groups/[id]/index.tsx` — Group detail

Nav bar: `← Back` chevron left, kebab right (contains contextual items depending on role — see below). Scrollable body:

1. **Header block:** Display-size group name, body-16 goal description (`--text-muted`), caption-row metadata: `{member_count} of 10 members · {Photo|Video} · {human_tz_label}`.
2. **Post-create banner (admin only, one-time):** `Group created — share the code to invite friends.` — dismissible, auto-hides after 8s, persistence via `expo-secure-store` `seen_create_banner:{group_id}`.
3. **Admin invite panel (admin only):** Card (`--surface`, `lg` radius, `e1-subtle`). Caption label: `Invite code`. `InviteCodeChip` with the active code. `Share code` PrimaryButton below chip. Below that, right-aligned accent-cyan link `Regenerate code`.
4. **Members section:** Caption label `Members ({n})`. Vertical list of member rows: Avatar (40pt) + name stack (display_name body-16, subtle `You` caption if it's the current user) + optional `ADMIN` badge pill right-aligned. Rows are non-tappable in P2 (member profiles are out of scope).
5. **Bottom destructive zone** (separated from members by `2xl` gap + 1px `--border` top):
   - Non-admin: `Leave group` DestructiveTextButton, centered.
   - Admin: `Transfer admin` accent-cyan text link (centered), `lg` gap, then `Delete group` DestructiveTextButton (centered).

**Kebab menu contents:**
- Non-admin: `Leave group` (destructive item — mirrors bottom button for reachability)
- Admin: `Regenerate code`, `Transfer admin`, `Delete group` (destructive last)

**Loading state:** Skeleton header (display-height block), skeleton invite panel (rectangle), skeleton member rows (3 circle+line placeholders). TanStack Query `isLoading` gates.

**Member-cap state:** When `member_count === 10`, caption below the members list reads `Your group's full — 10 is the cap for now.` (`--text-muted`, not destructive). Admin invite panel still renders; `Share code` still works (redemption attempts will fail server-side with `group_full` and the deep-link preview will show the error copy).

### `app/(app)/groups/join.tsx` — Join with code

Nav bar: `← Back` left, title `Got a code?` left. Body: Display header + subtitle copy (see Copywriting). Single TextInput with the invite-code treatment (see Typography): autocapitalize=`characters`, autocorrect=off, maxLength=9 (8 chars + 1 optional dash). Client-side normalization: strips non-alphanumeric, uppercases, inserts dash after 4 for display. Helper caption: `8 letters and numbers. Dashes optional.` Below input, sticky PrimaryButton `Join group` (disabled until 8 valid chars entered). Inline error rendering under input for all typed errors.

On success → `router.replace('/groups/{group_id}')`.

### `app/invite/[code].tsx` — Deep-link landing

**Three path variants** based on auth and membership state:

1. **Unauthenticated:** Call `get_invite_preview(code)` (SECURITY DEFINER — works without session). Render preview card: avatar (admin's) + "**{admin_name}** invited you" (display size) + "Join **{group_name}** — {member_count} already in." + PrimaryButton `Sign in to join` + ghost link `Create an account to join`. On button tap: persist code in `expo-secure-store` key `pending_invite_code`, route to `/login` (or `/signup`). Post-auth, the root layout checks for the key and routes back here.

2. **Authenticated, not already a member:** Same preview card, but CTA becomes `Join group` (primary). On tap: call `redeem_invite(code)`. On success → `router.replace('/groups/{group_id}')`. On typed error → render error copy inline below the preview card (destructive color) + Retry button. `already_member` path: show `already_member` copy + accent link `Open {group_name}` → `/groups/{group_id}`.

3. **Preview RPC fails / invalid code:** Render `Invite not found` (Heading 1) + body `We don't know that code. Double-check it with whoever invited you.` + ghost link `Back to groups` (→ `/`).

**Loading state:** Skeleton preview card while RPC resolves.

### Modals (all invoked from group-detail)

All modals use the P2 `Modal` primitive. **Every `cancelLabel` is context-specific per the copywriting contract.**

**Admin-leave branching modal** — invoked when admin taps "Leave group" (kebab or bottom button).
- Primary: `Transfer admin instead` → closes this modal, opens Transfer-admin picker.
- Secondary (destructive-text, stacked): `Delete the group` → closes this modal, opens Delete-group confirmation.
- Dismiss (`cancelLabel`): `Never mind`.

**Transfer admin picker modal** — Full P2 `Modal` with custom body: list of current group members (excluding self). Each row = Avatar + name + radio circle on the right. Tap selects. Selection highlights row with `--surface-muted` bg. Primary button enabled once a row is selected: `Transfer to {name}`. Dismiss (`cancelLabel`): `Keep my admin role`. On success → close modal + banner on group-detail: `{name} is now the admin.` Current user's destructive zone changes to non-admin `Leave group` only.

**Delete group confirmation** — P2 `Modal`: title `Delete this group?`, body copy per table above, primary destructive filled `Delete group`. Dismiss (`cancelLabel`): `Keep the group`. On success → `router.replace('/')` + caption toast `Group deleted.`

**Regenerate invite confirmation** — P2 `Modal`: title `Make a new code?`, primary `Regenerate` (primary, not destructive). Dismiss (`cancelLabel`): `Keep current code`. On success → close + new banner `New code ready — share it with anyone who hasn't joined yet.` + invite chip updates with new code.

**Member leave confirmation** — P2 `Modal`: title `Leave {group_name}?`, primary destructive `Leave group`. Dismiss (`cancelLabel`): `Stay in group`. On success → `router.replace('/')`, group disappears silently.

## Interaction Contracts

Inherited from Phase 1 plus P2-specific additions:

| Contract | Specification |
|----------|---------------|
| Tap feedback | P1 — scale 0.97, opacity 0.9, 150ms ease-out on all buttons and pressable rows |
| Group-list row press | Same as above; adds `--surface-muted` background during press |
| Copy code | Triggers `Clipboard.setStringAsync` + `Haptics.notificationAsync(Success)` + in-place `Copy` → `Copied ✓` text swap for 2s. No toast. |
| Share code | Triggers RN `Share.share({ message: ... })` with the D-19 pre-formatted message. No result handling — native sheet dismisses itself. |
| Code-entry normalization | On every keystroke: strip non-alphanumeric, uppercase, re-render with dash after char 4. Preserve caret position at the logical end. |
| Segmented control press | Scale 0.98 on active segment, instant swap, 100ms ease. No press animation on inactive segments beyond standard Pressable opacity. |
| Modal presentation | iOS: `presentationStyle="pageSheet"` for full-screen modals (timezone picker), standard `Modal` for centered confirmations. Android: standard `Modal` with `transparent={true}` + scrim view. |
| Modal scrim tap | Dismisses modal (calls `onDismiss`). Destructive modals (Delete group) should still scrim-dismiss — the primary button is the confirmation, scrim-tap = dismiss. |
| Modal dismiss labels | Every modal passes a context-specific `cancelLabel`; the generic word `Cancel` is banned by the copywriting contract and is dev-mode-warned by the `Modal` component. |
| Keyboard handling | `KeyboardAvoidingView` on create-group and join-with-code screens (inherited P1 pattern). Tapping outside dismisses keyboard. |
| Deep-link arrival | `app/invite/[code].tsx` is the Expo Router target for `accountibuzz://invite/*`. Root layout checks `pending_invite_code` in `expo-secure-store` post-auth and re-routes. |
| Auth detour | Before routing to `/login`, `await SecureStore.setItemAsync('pending_invite_code', code)`. On post-auth root-layout mount, if key exists, `SecureStore.getItemAsync` → `router.replace('/invite/{code}')` → clear key on successful redemption only (preserve on failure so user can retry). |
| Groups list refresh | Pull-to-refresh via FlatList `refreshControl`. Auto-refetch on screen focus via TanStack Query `refetchOnWindowFocus` equivalent. |
| Realtime member list (optional per CONTEXT D-discretion) | If shipped: subscribe to `group_members` inserts/deletes filtered by `group_id`. Animate additions (fade-in) and removals (fade-out) over 200ms. If not shipped in P2: pull-to-refresh is the update path. |
| Post-create banner dismiss | 8s auto-hide via `setTimeout`; × tap hides immediately. Persistence via `expo-secure-store` `seen_create_banner:{group_id}` — never shown again for same group. |
| Loading buttons | P1 pattern — label swaps for spinner, button disabled. Applies to all primary CTAs in P2. |
| Destructive button loading | Same pattern — spinner color `--primary-fg` for destructive filled buttons (red bg, near-black spinner). |

## Accessibility Contract

Inherited from Phase 1. P2-specific additions:

| Requirement | Specification |
|-------------|---------------|
| Invite code readability | `accessibilityLabel` on code chip reads the code letter-by-letter: `"Invite code: A, B, C, D, dash, E, F, 1, 2"` for screen readers. Otherwise VoiceOver reads "ABCD-EF12" as a word. |
| Segmented control | Each segment has `accessibilityRole="button"` and `accessibilityState={{ selected: boolean }}`. The whole control has `accessibilityLabel="Submission type"`. |
| Admin badge | Badge gets `accessibilityLabel="Admin"` — readers hear "ADMIN" not "A-D-M-I-N". |
| Modal | `accessibilityViewIsModal={true}` on sheet; title gets `accessibilityRole="header"`. Primary action gets `accessibilityRole="button"`. Dismiss button's `accessibilityLabel` is the context-specific `cancelLabel` (e.g. "Stay in group") — never a generic "Cancel". |
| Dismiss × icons | `accessibilityLabel="Close"` + `accessibilityRole="button"` + 44×44pt hit slop. |
| Member-cap state copy | Read by screen reader as part of the member list section — not hidden decoration. |
| Color-only cues forbidden | Error under code-entry is red border + red helper text + error icon (Feather `alert-circle`). Success (code-copied) is `--success` color + checkmark icon. |
| Dynamic Type | All copy scales with OS text-size. 140-char goal input must not clip at 200% type size — use `multiline` + vertical growth. |
