**UAT Checklist (record PASS/FAIL/DEFER for each in 04-VERIFICATION.md UAT Walkthrough section):**

CK-01 (LB-01 leaderboard renders) — On Device B, open group-detail. Confirm Leaderboard section renders top-5 (or all if <= 5) sorted by points DESC, current_streak DESC. Rank-1 chip is yellow (--primary); ranks 2-3 chips are surface-muted; rank 4+ is bare numeral.

CK-02 (LB-02 cross-device Realtime — HARD GATE, mirror of P3 CK-5) — On Device B, navigate to group-detail and stay on that screen. On Device A (admin), approve a pending submission for one of Device B's group members. Within ~1 second, Device B's leaderboard should reorder and the approved member's row should show updated points + current_streak. PASS = visible reorder within ~2s; FAIL = no update or > 5s lag. If FAIL: check via psql or Supabase SQL editor that `select tablename from pg_publication_tables where pubname = 'supabase_realtime' and schemaname = 'public'` includes group_members (CGF-1). If FAIL despite publication entry, check Realtime channel subscription succeeded (look at supabase-js logs in the dev client).

CK-03 (D-19 column-allowlist trigger) — From a developer console (psql or Supabase SQL editor with admin JWT), attempt `update public.group_members set points = 9999 where ...`. Expect raise: 'group_members counter columns are server-managed'. PASS = raise; FAIL = silent update.

CK-04 (FEED-01 today's feed renders) — On Device A, open the test group's group-detail. Confirm Today's posts section shows the approved submission(s) — avatar + name + relative time + 80x80 thumbnail + caption (if present). Video thumbnails autoplay muted+looping.

CK-05 (FEED-01 Realtime feed prepend) — On Device B, on group-detail. On Device A, approve a fresh pending submission. The feed on Device B should prepend the new FeedItem within ~1 second. Existing items shift down. PASS = prepend within ~2s.

CK-06 (FEED-02 still-to-post visible) — On Device A or B, open group-detail. Confirm Still to post section shows overlapping avatars + first-name comma list + cutoff inline (e.g. 9:00 PM cutoff) for members who have not submitted today. Section is ABSENT if everyone has submitted.

CK-07 (FEED-03 missed yesterday tombstones — TONE GATE) — Open group-detail. If at least one member missed yesterday (no approved submission for yesterday's local_date), confirm MISSED YESTERDAY section renders with surface-muted/40 background, 28pt avatars at opacity 0.7, comma-separated names, and trailing "Streaks reset at 12:00 AM PT." (or the group's tz short label). PASS = quiet, factual, not red, no judgmental language; FAIL = destructive color or "X lost their streak" copy. Section is ABSENT if no misses.

CK-08 (D-13 Today GroupCard social-signal) — On Today screen, confirm each GroupCard renders the social-signal line below the existing CTA: "{posted}/{total} posted · {points} pts · 🔥{streak}". When posted === 0, line shows "0/{total} posted · be the first". When streak === 0, the streak fragment renders with muted color but the flame stays.

CK-09 (D-15 per-card Realtime invalidation) — On Today screen, open the test group's card. On Device A, approve a submission for this group. Within ~1 second, Device B's GroupCard social-signal line should update (e.g. 4/6 posted -> 5/6 posted). PASS = visible update within ~2s.

CK-10 (D-16 GroupCard tap unchanged) — Tap a GroupCard on Today screen. Confirms it routes to group-detail (no new modal, no leaderboard expansion via Today). PASS = navigation works the same as P3.

CK-11 (D-10 leaderboard tap-to-expand) — On group-detail, if total members > 5, tap "See all" or the bottom expand button. Leaderboard expands in-place to show all members. Tap "Show top 5" — collapses. No new route, no modal.

CK-12 (Pitfall #11 Realtime cleanup on tab navigation) — Navigate to group-detail (Realtime channels subscribe). Tab to Profile. Tab back to Groups -> group-detail. Approve a submission on Device A. Confirm Device B's group-detail still receives the patch (channel re-subscribed correctly on focus). Then tab away from group-detail to Profile, leave the screen for ~30 seconds. Confirm no errors / channel-leak warnings in dev console.

CK-13 (D-11 fullscreen viewer — HARD GATE) — On Device A or B, open group-detail and navigate to a Today's posts FeedItem. Tap the 80×80 thumbnail. The shared `<MediaViewer>` Modal should open at full bleed: photo renders centered with `contentFit="contain"` on a black background; video renders with native controls + sound enabled. Tap the close button (top-right "✕"). Modal dismisses. Hardware back button on Android also dismisses. PASS = viewer mounts on tap + closes correctly; FAIL = no-op / nothing happens / app crashes. **Mirrors UI-SPEC line 546 binding contract.**

CK-14 (LayoutAnimation row-reorder + 250ms cross-fade — visual confirmation, soft gate) — On Device B, on group-detail. On Device A, approve a submission for a member who is currently mid-pack on the leaderboard (e.g. rank 3 with points that will overtake rank 2 after the +1). The leaderboard rows should animate position swap (~200ms ease-in-ease-out via LayoutAnimation), AND the affected row's points + streak number should briefly cross-fade (250ms). Toggle iOS Settings → Accessibility → Motion → Reduce Motion = ON, return to the app, repeat the test: animations should be SKIPPED (instant snap). PASS = visible animation when reduce-motion OFF + instant snap when ON; SOFT FAIL = animation absent in either path (deferred polish task; document in Outstanding Items).
