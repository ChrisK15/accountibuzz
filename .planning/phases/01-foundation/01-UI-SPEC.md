# Phase 1 — UI Design Contract

> Visual and interaction contract for Foundation phase (auth + profile).
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Source of truth for visual tokens: `design_refs/` (Lovable-generated, user-supplied 2026-04-21).

## Design System

| Property | Value |
|----------|-------|
| Tool | none (React Native project — shadcn is web-only and not applicable) |
| Preset | not applicable |
| Component library | Hand-rolled RN primitives in `src/components/` + `expo-image` for avatars. No third-party UI kit in P1. |
| Icon library | `@expo/vector-icons` (bundled with Expo SDK 55; Feather + Ionicons subsets) |
| Font | **Manrope** (variable font via `expo-font` + `@expo-google-fonts/manrope`) |

**Theming:** Light + dark themes both shipped in P1 (the Lovable refs show both; OS-level `useColorScheme` drives selection). Tokens are defined as semantic names (`--primary`, `--surface`, etc.) with values resolved per theme.

## Spacing Scale

Declared values (from `design_refs/design_token_2.png` — user-locked):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, tight inline padding |
| sm | 8px | Compact element spacing (label → input) |
| md | 12px | Input internal padding, small gaps between related elements |
| lg | 16px | Default element spacing (between form fields, between stacked cards) |
| xl | 24px | Section padding, screen horizontal padding |
| 2xl | 32px | Major section breaks, header-to-content gap |

**Exceptions:**
- Minimum tap target = 44×44pt per Apple HIG / 48dp per Material — applies to icon-only controls (avatar edit pencil, back chevron). Buttons in the contract are already ≥48pt tall so no exception needed there.
- The user's scale includes **12** (not a pure 4/8/16 progression) — this is an intentional user decision from Lovable tokens and is accepted as the locked contract. All declared values are still multiples of 4.

## Typography

Font family: **Manrope** (variable). Declared roles from `design_refs/design_token_2.png`:

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 32px | 800 (ExtraBold) | 1.15 (~37px) | Screen hero titles ("Welcome back", "Let's set up your profile", "Join the buzz") |
| Heading 1 | 24px | 700 (Bold) | 1.25 (~30px) | Section headers (not used prominently in P1 but reserved) |
| Heading 2 | 20px | 700 (Bold) | 1.3 (~26px) | Secondary section headers, modal titles |
| Body | 16px | 500 (Medium) | 1.5 (~24px) | All body copy, button labels, input text, form values |
| Caption | 13px | 500 (Medium) | 1.4 (~18px) | Helper text under inputs, metadata, "Email can't be changed here." |

**Weights used across the whole phase:** 500 (Medium) for body/UI, 700 (Bold) for headings, 800 (ExtraBold) for display. Three weights total — declared because Manrope's variable axis makes the extra weight free at runtime and the Lovable contract specifies it.

**Tabular numbers:** Use `font-variant-numeric: tabular-nums` (RN: `fontVariant: ['tabular-nums']`) for any numeric displays (e.g. streak counter, point count — not in P1 but established now).

## Color

Tokens from `design_refs/design_token_1.png` + `token_values.docx`. All named with semantic CSS-var-style identifiers for theme swapping.

### Brand

| Role | Light value | Dark value | Usage |
|------|-------------|------------|-------|
| `--primary` | `#FFDE42` | `#FFDE42` | Primary CTA buttons ("Log in", "Create account", "Send reset link", "Reset password", "Continue", "Save changes"). Logo highlight ("buzz" pill). |

| Role | Value | Usage |
|------|-------|-------|

### Neutrals & Surfaces

| Role | Light | Dark | Usage |
|------|-------|------|-------|
| `--background` | `hsl(var(--background))` → `hsl(30 30% 97%)` warm off-white | `hsl(220 20% 6%)` near-black | Screen background |
| `--surface` | `hsl(var(--surface))` → `#FFFFFF` | `hsl(220 18% 10%)` | Cards, input fields |
| `--surface-muted` | `hsl(220 14% 92%)` (N-low) | `hsl(220 15% 14%)` | Disabled input background, subtle card variant |
| `--border` | `hsl(220 14% 88%)` | `hsl(220 12% 20%)` | Default input border, divider |
| `--text` | `hsl(220 15% 15%)` | `hsl(30 20% 95%)` | Primary body text |
| `--text-muted` | `hsl(220 10% 45%)` (N-mid darkened for contrast) | `hsl(220 10% 65%)` | Helper text, labels, captions |
| `--text-strong` | `hsl(220 15% 10%)` | `hsl(0 0% 100%)` | Display/heading text |

### Accent Reserved-For List (10% rule — EXPLICIT)

`--accent` (cyan `#53CBF3`) is used ONLY for:
1. Inline text links ("Forgot password?", "Sign up", "Log in" from signup, "Back to login", "Terms", "Privacy Policy")
2. "Add a photo" / "Change avatar" text button (ghost variant)
3. Focused text input 2px border ring
4. "Edit" text-button in profile top-right nav

`--accent` is NEVER used for: primary CTAs, body text, icons-by-default, backgrounds, success/confirmation signals, or decoration.

### 60/30/10 Split

- **60% dominant:** `--background` (warm off-white in light, near-black in dark)
- **30% secondary:** `--surface` + `--surface-muted` (input fields, cards like the "Today's goal" card on profile view)
- **10% accent:** `--primary` (yellow CTAs + logo highlight) and `--accent` (cyan links/focus) combined — yellow dominates the 10%, cyan is a thin sub-accent for navigational text

### Destructive Reserved-For List

`--destructive` (red) is used ONLY for:
1. "Log out" text-button on profile edit screen (appears as red text, not a filled button, because logout is destructive-adjacent but confirmable)
2. Inline form error messages ("We couldn't find an account with that email.", "Enter a valid email address.")
3. Input border when field has a validation error
4. Future "Delete account" filled button (not in P1 but declared in component inventory)

## Interaction Contracts

| Contract | Specification |
|----------|---------------|
| Tap feedback | All buttons scale to 0.97 + reduce opacity to 0.9 during press (150ms ease-out). RN: `Pressable` with `style={({pressed}) => ...}`. |
| Loading state | Primary button label swaps for centered spinner (same yellow bg, near-black spinner). Button disabled during load. |
| Focused input | 2px cyan border replaces default border. Keyboard shortcut: `Enter`/`Next` moves to next field; `Enter` on last field submits form. |
| Form submit | Submit disabled until Zod schema validates (RHF `formState.isValid`). Button shows loading spinner on submit. |
| Toast / feedback | Native iOS/Android toast equivalent not used in P1 — errors render inline under the field. Successful actions (e.g. reset link sent) navigate to a confirmation screen or render inline success copy below the CTA. |
| Keyboard handling | `KeyboardAvoidingView` on every auth screen. Tapping outside input dismisses keyboard. |
| Avatar tap (edit) | Opens `expo-image-picker` library. Camera-roll selection acceptable for avatars (per CONTEXT.md D-11). |
| Logout tap | Native `Alert.alert` confirmation (see Destructive Action Confirmations above). |
| Deep-link arrival (reset password) | App opens directly to `app/(auth)/reset-password` with token in URL params. If token invalid/expired → show error copy + "Request a new one" link back to forgot-password. |

## Accessibility Contract

| Requirement | Specification |
|-------------|---------------|
| Minimum tap target | 44×44pt (iOS) / 48dp (Android). Icon-only controls must meet this even if visually smaller. |
| Text contrast | Body text on background ≥ 4.5:1 (WCAG AA). Verified for both themes. Primary-yellow CTA uses near-black label to hit ~14:1. |
| `accessibilityLabel` | Every interactive non-text element (avatar edit, chevron back, logo if tappable) has an explicit label. |
| `accessibilityRole` | Buttons use `role="button"`; links use `role="link"`; inputs auto-inferred from `TextInput`. |
| Dynamic Type | Typography scales with OS text-size setting. Use RN's `allowFontScaling={true}` (default). Test at 200% text size — layouts must not clip. |
| Color-only cues forbidden | Error state is red border AND red helper text (never border alone). Focus state is cyan ring AND the label color remains readable. |
| Dark mode | Full parity with light. Follows OS `useColorScheme`. No manual toggle in P1. |

## Screen-by-Screen Contract

### `app/(auth)/login`
Logo (top) → Display title "Welcome back" → Body subtitle → Email input → Password input → "Forgot password?" right-aligned accent link → Primary CTA "Log in" → Centered ghost link "New here? Sign up"

### `app/(auth)/signup`
Logo → Display "Join the buzz" → Body "Three friends, one daily goal." → Email → Password (placeholder: "At least 8 characters") → Confirm password → Primary CTA "Create account" → Centered caption with Terms/Privacy accent links → Centered ghost link "Already have an account? Log in"

### `app/(auth)/forgot-password`
Logo → Display "Forgot password?" → Body subtitle → Email (may render in error state with "We couldn't find an account with that email.") → Primary CTA "Send reset link" → Centered ghost link "Remembered it? Back to login"

### `app/(auth)/reset-password`
Logo → Display "Set a new password" → Body "Choose something memorable but strong." → New password → Helper text "Use 8+ characters with a number and a symbol." → Confirm new password → Primary CTA "Reset password"

### Profile onboarding (soft-gate landing after signup — same `app/(app)/profile` but in "onboarding" visual state when `display_name === ''`)
Display "Let's set up your profile" (left-aligned) → Body subtitle → Centered purple avatar placeholder with `?` → Ghost link "Add a photo" below avatar → Display name input with placeholder "What should we call you?" → Sticky-bottom Primary CTA "Continue" → Ghost link below "Skip for now"

### `app/(app)/profile` (view state)
Nav bar: "Profile" left, "Edit" accent-cyan text-button right → Centered avatar (circle, AR initials on purple if no image) → Display name (Heading 1) → Email muted → Two stat cards side-by-side (`🔥 12 DAY STREAK` with yellow-wash bg, `2,340 POINTS` with cyan-wash bg) — stat cards are declared P1 shells, data arrives in P4 → "TODAY'S GOAL" card with goal text + yellow progress bar (declared shell, P4 will hydrate)

> **Note on stat cards in P1:** Lovable ref shows populated stats, but Phase 1's DB has no streak/points data yet (P4 fills triggers). The P1 implementation must render the card shells with placeholder zero-state (e.g. `—` or `0`) so the visual layout is locked but doesn't lie. The "Today's goal" card should be hidden in P1 (no group, no goal) — hide entire card if user has no groups.

### `app/(app)/profile` (edit state)
Nav bar: "Edit profile" left, "Cancel" accent-cyan text-button right → Centered avatar (tappable) → Ghost link "Change avatar" → Display name input (focused by default on entry) → Email input (disabled + muted) → Caption "Email can't be changed here." → Spacer → Sticky-bottom Primary CTA "Save changes" → Below CTA: centered destructive text-button "Log out"

## Design Token Export Shape

For the executor — tokens should land in `src/theme/tokens.ts` as a typed object, consumed by a `ThemeProvider` that picks light/dark via `useColorScheme()`:

```ts
// src/theme/tokens.ts (shape only — full values in contract above)
export const spacing = { xs: 4, sm: 8, md: 12, lg: 16, xl: 24, '2xl': 32 } as const;
export const radii   = { sm: 6, md: 12, lg: 20, pill: 9999 } as const;
export const fonts   = { display: { size: 32, weight: '800', lineHeight: 37 }, /* ... */ } as const;
export const colors = {
  light: { primary: '#FFDE42', accent: '#53CBF3', /* ... */ },
  dark:  { primary: '#FFDE42', accent: '#53CBF3', /* ... */ },
} as const;
```

Reference `design_refs/design_token_1.png` and `design_refs/design_token_2.png` as the authoritative visual artifact. If a token value in this contract and the PNG disagree, the **PNG wins** — update this doc.
