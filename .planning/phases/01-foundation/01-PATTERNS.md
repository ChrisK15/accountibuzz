### `app/_layout.tsx`

**Pattern source:** RESEARCH.md §"Pattern 2: Root Layout with Session-Aware Redirect" (lines 379-426)
**Canonical origin:** https://docs.expo.dev/router/reference/authentication/

**Core pattern to copy verbatim (RESEARCH.md lines 385-425):**

```typescript
import { useEffect, useState } from 'react';
import { Stack, useRouter, useSegments } from 'expo-router';
import { Session } from '@supabase/supabase-js';
import { QueryClient, QueryClientProvider } from '@tanstack/react-query';
import { supabase } from '../src/lib/supabase';

const queryClient = new QueryClient();

function useProtectedRoute(session: Session | null) {
  const segments = useSegments();
  const router = useRouter();

  useEffect(() => {
    const inAuthGroup = segments[0] === '(auth)';
    if (!session && !inAuthGroup) router.replace('/(auth)/login');
    else if (session && inAuthGroup) router.replace('/(app)/profile');
  }, [session, segments]);
}

export default function RootLayout() {
  const [session, setSession] = useState<Session | null>(null);

  useEffect(() => {
    supabase.auth.getSession().then(({ data }) => setSession(data.session));
    const { data: listener } = supabase.auth.onAuthStateChange((_e, s) =>
      setSession(s)
    );
    return () => listener.subscription.unsubscribe();
  }, []);

  useProtectedRoute(session);

  return (
    <QueryClientProvider client={queryClient}>
      <Stack screenOptions={{ headerShown: false }} />
    </QueryClientProvider>
  );
}
```

**Files that follow this layout pattern (minus redirect logic):** `app/(auth)/_layout.tsx`, `app/(app)/_layout.tsx` — each renders a `<Stack />` with default screen options.

### `supabase/migrations/0001_foundation.sql`

**Pattern source:** RESEARCH.md §"Pattern 3: `profiles` Auto-Insert Trigger" (lines 428-470) + ARCHITECTURE.md (RLS matrix, full schema)
**Canonical origin:** https://supabase.com/docs/guides/auth/managing-user-data

**Table creation pattern — every `create table` MUST be followed immediately by `enable row level security` + at least one policy (from RESEARCH.md Pitfall 1 + PITFALLS.md §3):**

```sql
create table public.profiles (
  id uuid primary key references auth.users(id) on delete cascade,
  display_name text not null default '',
  avatar_path text,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

alter table public.profiles enable row level security;

create policy "profiles_select_own_or_shared_group" on public.profiles
  for select using (
    id = auth.uid()
    -- group-shared visibility is added by later phases; P1 sees only own row
  );

create policy "profiles_update_own" on public.profiles
  for update using (id = auth.uid())
  with check (id = auth.uid());
```

**`security definer` trigger pattern (RESEARCH.md lines 456-467 + Pitfall 4):**

```sql
create or replace function public.handle_new_user()
returns trigger language plpgsql security definer as $$
begin
  insert into public.profiles (id) values (new.id);
  return new;
end;
$$;

create trigger on_auth_user_created
  after insert on auth.users
  for each row execute function public.handle_new_user();
```

**Full-schema scope (CONTEXT.md D-01):** Tables `profiles`, `groups`, `group_members`, `submissions`, `invites`, `notifications_outbox` — all shipped in this single migration, all with RLS enabled and at least one policy each. **Storage bucket + policies (CONTEXT.md D-03 + D-11, RESEARCH.md Pitfall 6 lines 670-697):**

```sql
-- avatars bucket (public-read, path-gated write)
insert into storage.buckets (id, name, public) values ('avatars', 'avatars', true);

create policy "avatars_select_public" on storage.objects
  for select using (bucket_id = 'avatars');

create policy "avatars_insert_own" on storage.objects
  for insert to authenticated
  with check (
    bucket_id = 'avatars'
    AND (storage.foldername(name))[1] = auth.uid()::text

create policy "avatars_update_own" on storage.objects
  for update to authenticated

    bucket_id = 'avatars'
    AND (storage.foldername(name))[1] = auth.uid()::text

-- submissions bucket (private, path-encoded group_id/user_id/date)

-- policies referencing is_group_member helper — see ARCHITECTURE.md
```

### `.github/workflows/rls-check.yml`

**Pattern source:** RESEARCH.md §"Pattern 4: RLS CI Probe" (lines 472-533)
**Canonical origin:** CONTEXT.md D-04

**Core pattern — boot local Supabase + apply migrations + SQL probe:**

```yaml
name: RLS Check
on: [push, pull_request]
jobs:
  rls:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: supabase/setup-cli@v1
        with: { version: <PIN_SPECIFIC_VERSION> }  # Pitfall 7: pin explicitly
      - run: supabase start
      - run: supabase db reset  # applies migrations + seed
      - name: Probe RLS status
        run: |
          OFF=$(supabase db execute --db-url "..." <<'SQL'
            SELECT format('%I.%I', schemaname, tablename)
            FROM pg_tables
            WHERE schemaname = 'public' AND rowsecurity = false;
          SQL
          )
          if [ -n "$OFF" ]; then
            echo "::error::Tables without RLS: $OFF"
            exit 1
          fi
```

**Storage-policy probe (RESEARCH.md lines 523-532 — the `pg_policies` variant is more portable than `storage.objects_policies`):**

```sql
SELECT b.id AS bucket
FROM storage.buckets b
WHERE b.id IN ('submissions', 'avatars')
  AND NOT EXISTS (
    SELECT 1 FROM pg_policies
    WHERE schemaname = 'storage' AND tablename = 'objects'
      AND qual LIKE '%' || b.id || '%'
  );
```

**Wave 0 validation required (RESEARCH.md Assumption A1):** The exact SQL shape for the storage-policy probe must be confirmed against a live `supabase start` instance before merge. Failure mode is loud (build fails), not silent.

**CLI version pinning (RESEARCH.md Pitfall 7):** Pin `supabase/setup-cli` to a specific CLI release; never `latest`.

## Shared Patterns (cross-cutting, apply to multiple files)

### Shared Pattern 1: RLS-by-default (applies to every `create table`)

**Source:** RESEARCH.md Pitfall 1 (lines 623-638) + PITFALLS.md §3
**Apply to:** every table definition in `supabase/migrations/0001_foundation.sql` and every future migration.

**Invariant:** Every `create table public.X` is followed — **in the same migration, adjacent in the file** — by `alter table public.X enable row level security;` and at least one `create policy` statement. No exceptions.

**Enforcement:** `.github/workflows/rls-check.yml` fails the build on any `pg_tables.rowsecurity = false` row in `public` schema.

### Shared Pattern 2: Supabase singleton import (applies to every client file that talks to Supabase)

**Source:** RESEARCH.md §"Don't Hand-Roll" + Pattern 1
**Apply to:** every file in `src/features/**`, `app/**` that calls auth/storage/postgres.

**Invariant:** Import the singleton from `src/lib/supabase.ts`. Never call `createClient` anywhere else.

```typescript

```

### Shared Pattern 3: RHF + Zod for every form (applies to all auth + profile screens)

**Source:** RESEARCH.md §"Don't Hand-Roll" row 2 + Code Examples lines 712-753
**Apply to:** `login.tsx`, `signup.tsx`, `forgot-password.tsx`, `reset-password.tsx`, `profile.tsx`.

**Invariant:** Every form = Zod schema (in a `schemas.ts` file next to the feature) + `useForm({ resolver: zodResolver(schema) })` + `<Controller>` for each input. No manual `useState` per field, no hand-coded validators.

### Shared Pattern 4: Error handling via supabase-js `{ data, error }` destructure

**Source:** Canonical supabase-js API (used in every snippet in RESEARCH.md).
**Apply to:** every `supabase.auth.*`, `supabase.from(...).*`, `supabase.storage.*` call.

**Invariant:**

```typescript
const { data, error } = await supabase.something(...);
if (error) {
  // surface via toast / form error — do NOT swallow
  return;
}
// use data
```

No try/catch around supabase-js calls unless wrapping an `await`-heavy pipeline (e.g., avatar upload) where a thrown upload error propagates up.

### Shared Pattern 5: TanStack Query v5 API (`isPending`, not `isLoading`)

**Source:** RESEARCH.md §"State of the Art" table — "TanStack Query `isLoading` → `isPending` (v5, 2024)"
**Apply to:** every hook in `src/features/**/use*.ts`.

**Invariant:** Use v5 naming — `isPending`, `isError`, `error`, `data`. Ignore tutorials that still reference `isLoading`.

### Shared Pattern 6: Environment variables via `EXPO_PUBLIC_*` prefix only (for client)
