    # Accountibuzz

    A missed day is visible to your group within hours — the social cost of skipping is what keeps you accountable.

    ## Stack
    - Expo SDK 55 (RN 0.83.1, React 19.2, New Arch only)
    - Supabase (Postgres + Auth + Storage + Realtime + Edge Functions + pg_cron)

    ## Prerequisites
    - Node 20+
    - Docker Desktop (for `supabase start`)
    - Supabase CLI (`npm install -D supabase` — already a dev dep)
    - Expo Go app on a physical iOS or Android device
    - Xcode (macOS) OR Android Studio for simulators

    ## First-time setup
    1. `npm install`
    2. Create a Supabase project at https://supabase.com/dashboard
    3. Copy `.env.example` → `.env` and paste:
       - `EXPO_PUBLIC_SUPABASE_URL` from Project Settings → API → Project URL
       - `EXPO_PUBLIC_SUPABASE_ANON_KEY` from Project Settings → API → anon/public key
    4. In Supabase Dashboard → Authentication:
       - Providers → Email → **Disable "Confirm email"** (D-09)
       - URL Configuration → Add `accountibuzz://reset-password` to Redirect URLs
    5. Link to your project: `npx supabase link --project-ref <your-ref>`
    6. Apply the schema: `npx supabase db push`

    ## Local dev loop
    - `npm run start` — Expo dev server + QR code for Expo Go
    - `npm run ios` / `npm run android`
    - `npm test` — Jest (client)
    - `npm run test:all` — Jest + pgTAP (`supabase test db`)
    - `npx supabase db reset` — reapply migrations + seed locally
    - `npm run types:gen` — regenerate DB types after any migration

    ## RLS CI check
    `.github/workflows/rls-check.yml` boots a local Supabase stack, applies migrations, and probes `pg_tables` for any public table with `rowsecurity=false`. It also confirms `avatars` + `submissions` buckets have policies. Build fails on any violation.

    ## Project layout
    - `app/` — Expo Router routes (`(auth)`, `(app)`)
    - `src/lib/` — Supabase client singleton + encrypted session storage adapter
    - `src/features/` — auth + profile feature modules
    - `src/components/` — hand-rolled RN UI primitives
    - `src/theme/` — tokens + ThemeProvider
    - `supabase/migrations/0001_foundation.sql` — full schema + RLS + helpers
    - `supabase/tests/*.sql` — pgTAP RLS + trigger tests

    ## Planning artifacts
    See `.planning/` for phase plans, research, and context. `gsd-*` commands drive the workflow.
