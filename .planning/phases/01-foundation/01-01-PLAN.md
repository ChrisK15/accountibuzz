<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Device keystore ↔ JS heap | AES encryption key lives in SecureStore (iOS Keychain / Android Keystore); session ciphertext in AsyncStorage |
| Client app ↔ Supabase Auth (GoTrue) | Anon key + email/password over TLS 1.2+ |
| Client bundle ↔ env vars | `EXPO_PUBLIC_*` vars ship in bundle; only anon key is allowed there |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | Session token at rest in AsyncStorage | mitigate | AsyncStorage value is AES-CTR ciphertext; key lives in `expo-secure-store` (OS keystore). Implemented by `LargeSecureStore` (RESEARCH.md Pattern 1). [CITED: supabase.com/blog/react-native-authentication] |
| T-01-02 | Elevation of Privilege | Service role key accidentally shipped in RN bundle | mitigate | Only `EXPO_PUBLIC_SUPABASE_URL` and `EXPO_PUBLIC_SUPABASE_ANON_KEY` in `.env.example`; lint/grep invariant in later plans blocks `SERVICE_ROLE` in `src/`/`app/`. `.gitignore` excludes `.env`. |
| T-01-03 | Spoofing / auth bug | `detectSessionInUrl: true` default parses bogus URLs on RN | mitigate | Explicit `detectSessionInUrl: false` in `createClient` config (Pitfall 2). Enforced by `supabase-client.test.ts` and `<acceptance_criteria>` grep. |
| T-01-04 | Denial of Service | Background token-refresh hammers backend / drains battery | mitigate | `AppState` listener calls `stopAutoRefresh` on background, `startAutoRefresh` on foreground (Pitfall 3). |
| T-01-05 | Tampering | Weak RNG for AES key on RN (no native `crypto.getRandomValues`) | mitigate | `react-native-get-random-values` polyfill imported as the FIRST line of `src/lib/supabase.ts`. |
| T-01-06 | Information Disclosure | Env var file committed to repo | mitigate | `.env` in `.gitignore`; only `.env.example` with placeholder values is tracked. |
</threat_model>
