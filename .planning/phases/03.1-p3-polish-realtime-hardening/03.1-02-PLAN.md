<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client (RN) → Expo Router navigation | Untrusted user gestures (swipe-dismiss); the modal disables gesture-dismiss to prevent accidental cancellation of a partially-completed capture |
| Tabs context → nested Stack context | Stack-only screenOptions are isolated to the nested layout; cannot leak back into the parent Tabs animation registry (the original P3 crash root cause) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-031-06 | Tampering / DoS (regression) | Parent Tabs `sceneStyleInterpolator` animation registry | mitigate | The nested Stack layout isolates the modal screenOptions from the Tabs context — restoring the originally-spec'd presentation WITHOUT triggering the original P3 inline-fix crash. Acceptance criterion (Task 1) enforces parent `app/(app)/_layout.tsx` Tabs.Screen registration UNCHANGED. UAT step (Task 3) verifies no `sceneStyleInterpolator undefined` error appears in Metro logs. |
| T-031-07 | User Error / Accidental Data Loss | gestureEnabled on capture modal | mitigate | `gestureEnabled: false` in the nested Stack screenOptions disables the iOS swipe-down dismiss gesture. Prevents accidental cancellation of a partially-completed capture (e.g., user is composing a caption when an inadvertent downward swipe would dismiss the modal and discard the photo URI from in-memory state). UAT step 6-9 (Task 3) is the verification gate. |
| T-031-08 | Information Disclosure | Capture-screen file dead code | accept | The inert `stackScreen` const + 5 JSX references are dead code, not security-sensitive. Removing them improves readability and removes a "the codebase has admitted defeat on modal presentation" comment trail that could mislead future contributors. No security threat — disposition is `accept` (defense-in-depth: cleaner code is easier to audit). |
</threat_model>
