# Phase 03.1 Plan 03: Shutter Visual Rewrite Summary

**One-liner:** Replaced the user-flagged "square inside a circle" Shutter geometry with a Feather-icon-based 3-variant design (camera/video glyphs + clean filled red recording circle), preserving the public ShutterProps API so the capture screen needs no edits.

## What Shipped

### `src/components/Shutter.tsx` (rewritten)

The 3-nested-shape recording variant (4pt outer ring + 44pt red square with `borderRadius: 0` + 16pt white center square — the source of the 2026-05-06 user complaint) was replaced with a uniform 52pt inner circle that hosts a Feather glyph for idle states or stays clean for recording:

| Variant            | Inner-fill geometry                       | Glyph                            |
| ------------------ | ----------------------------------------- | -------------------------------- |
| `photo`            | 52pt theme-primary (yellow) circle        | Feather `camera`, 30pt, white    |
| `video-idle`       | 52pt theme-primary (yellow) circle        | Feather `video`, 30pt, white     |
| `video-recording`  | 52pt destructive-red filled circle        | (none — clean filled circle)     |

All variants share the same outer 72pt 4pt white ring with the 1.4s opacity pulse animation (preserved verbatim from the previous implementation). The `Pressable`, `accessibilityLabel`, and `ShutterProps` exports are unchanged.

### `tests/components/Shutter.test.tsx` (new — 4 cases)

Behavior-preservation safety net mirroring `tests/components/StatusPill.test.tsx`:
1. Photo variant renders with "Take photo" accessibility label
2. Video-idle variant renders with "Start recording" accessibility label
3. Video-recording variant renders with "Stop recording" accessibility label
4. Pressing the Shutter fires onPress exactly once

All 4 tests pass against both the pre-rewrite implementation (gate: behavior preservation confirmed) and the post-rewrite implementation (gate: rewrite did not regress accessibility/onPress contract).

### `03.1-VERIFICATION.md` (new — UAT scaffold)

Created the phase-wide verification template with Plan 03's CK-09..CK-12 receipts wired up. Plans 01 and 02 append their CK-01..CK-08 sections in-place when their UAT gates land.
