## Plan 03 — Shutter Visual Rewrite (D-08..D-10)

**What shipped:**
- `src/components/Shutter.tsx` rewritten — Feather `camera` glyph for `photo`, Feather `video` glyph for `video-idle`, clean filled red circle for `video-recording`. Outer-ring opacity pulse preserved. Public `ShutterProps` API unchanged.
- `tests/components/Shutter.test.tsx` — 4-case behavior-preservation safety net (3 variants + onPress firing). All GREEN against the rewrite.

**Closes:** `.planning/notes/2026-05-06T01-05-10Z.md` ("fix record button UI it looks bad — very blocky, square inside a circle").

### How to verify (iOS dev client primary, Android optional)

1. **Setup**
   - Build + install latest dev client with the rewrite deployed
   - Sign in to a group where the user is a member
   - Have one PHOTO group and one VIDEO group available (or a video-capable group with mode toggling)

2. **Walkthrough — photo variant (CK-09)**
   - Navigate to capture screen for a PHOTO group
   - Observe Shutter at the bottom
   - PASS criterion: 72pt circle, 4pt white outer ring, 52pt theme-primary (yellow) inner circle with white Feather `camera` glyph centered. Should NOT read as "blocky" or "square inside a circle"
   - Capture a photo by tapping — confirm onPress still fires (preview/upload flow proceeds)

3. **Walkthrough — video-idle variant (CK-10)**
   - Navigate to capture screen for a VIDEO group (or toggle to video mode)
   - Observe Shutter — should match the photo variant geometry except the inner circle now contains a Feather `video` glyph (filmstrip icon)
   - PASS criterion: clean circle-inside-circle with a video glyph centered

4. **Walkthrough — video-recording variant (CK-11)**
   - From video-idle, tap the Shutter to start recording
   - Observe transition to recording state
   - PASS criterion: 72pt outer ring (4pt white border with 1.4s opacity pulse 0.85↔1.0) + 52pt SOLID FILLED RED CIRCLE inside it. NO inner glyph. NO inner white square. NO 90-degree corners. Should read as "ring + red dot"
   - Confirm outer ring opacity pulse is visibly animating

5. **Walkthrough — recording → stopped**
   - Tap Shutter again to stop recording
   - Observe transition back to video-idle state with the Feather video glyph

6. **Walkthrough — accessibility labels (CK-12)**
   - Enable VoiceOver (iOS) or TalkBack (Android)
   - Focus on Shutter at each variant — should announce "Take photo" / "Start recording" / "Stop recording"
   - PASS criterion: all 3 labels announced correctly

- Recording variant still shows nested square → `borderRadius: 0` was not removed. Check `grep "borderRadius: 0" src/components/Shutter.tsx`.
- Glyph not visible → Feather not bundled OR `color="#FFFFFF"` missing. Check the import + Feather usage line.
- Glyph too small / too large → `ICON_SIZE` outside 28-34 range. Adjust to taste; if 30 reads too small try 34, if too large try 28.
- Glyph poorly chosen → swap per Claude's discretion (D-09 lists `aperture` / `circle` / `play-circle` as alternates if `camera` / `video` don't render well at the final size).
