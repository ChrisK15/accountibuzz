**Walkthrough (per Plan 03.1-01 Task 5 `<how-to-verify>`):**

Two physical iOS devices on the same WiFi.

Setup:
- Device A: signed in as the admin of a test group
- Device B: signed in as a regular member of the same test group
- Both devices on the latest dev client build with the new hook deployed (rebuild required after this plan's commits land on `main`)

Walkthrough — badge auto-increment (group-detail):
1. Device A: open the group, land on group-detail. Note the "Pending Review (N)" badge value (likely 0 if no pending submissions). Stay on group-detail.
2. Device B: tap "Capture", record/take a fresh photo or video, submit. Observe Device B's status pill flip to "pending".
3. Device A: WITHOUT navigating away, observe the "Pending Review (N)" badge — it should increment by 1 within ~2 seconds.

Walkthrough — queue list auto-prepend (review-screen):
4. Device A: tap the "Pending Review (N)" badge to navigate to `/groups/[id]/review`. Note the queue list. Stay on review-screen.
5. Device B: submit ANOTHER fresh submission (use a second test member account if needed, or have admin's group be 3+ members).
6. Device A: WITHOUT navigating away, observe the queue list — the new pending row should appear within ~2 seconds, with the correct member's display_name + avatar.

Walkthrough — count-decrement on review (back to badge):
7. Device A: approve or reject one submission from the swipe queue.
8. Device A: navigate back to group-detail. Observe the "Pending Review (N)" badge — it should reflect the new (decremented) count immediately.

Negative test — non-admin device does NOT subscribe:
9. On Device B (regular member), open Metro logs (Expo DevTools → Logs). Navigate to the group's group-detail screen.
10. Observe Metro logs — there should be ZERO log entries containing `review-queue:` for Device B.
