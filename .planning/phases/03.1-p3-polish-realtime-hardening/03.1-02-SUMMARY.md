# Plan 03.1-02 — Capture Screen Modal Presentation Restore

## What shipped

A nested `<Stack>` layout under the capture route that re-applies the
`presentation: 'modal'` semantics commit `298fe64` ("Capture screen crash on
open") accidentally stripped from the Tabs context while fixing the
`sceneStyleInterpolator` runtime crash. The nested-Stack pattern is the
React-Navigation-recommended escape hatch for "Stack-only options on a Tabs
screen": Tabs.Screen registration stays vanilla (no invalid options), and the
nested Stack inside the route group owns the modal presentation contract.

## Decisions applied

- **D-05 (presentation contract)** — `presentation: 'modal'` lives on the
  nested Stack inside `app/(app)/capture/_layout.tsx`, NOT on the parent
  Tabs.Screen registration. This is the only shape that satisfies BOTH "no
  `sceneStyleInterpolator` crash on open" AND "slides up from bottom like a
  modal".
- **D-06 (parent Tabs untouched)** — `app/(app)/_layout.tsx` Tabs.Screen for
  `capture` still uses only `href: null` (kept hidden from the tab bar; no
  Stack-only options leaked into the Tabs context). The chore commit also
  removed the residual `stackScreen` const from `[groupId].tsx` left over from
  the original crash-fix attempt.
- **D-07 (interaction model)** — `gestureEnabled: false` (no swipe-down
  dismissal — capture must close via explicit "X"/cancel to avoid losing a
  half-recorded clip), `headerShown: false` (the screen draws its own header).
